## 2. Logging & Auditing Audit

### What is covered

- Every HTTP request logs entry (`Incoming request: METHOD /path`) and exit (`Request done: METHOD /path → STATUS (Xms)`) via middleware in both services.
- All business events log at INFO: event processing start, Account Service call result, idempotency hits, failures.
- All log lines include `trace_id`, `timestamp`, `level`, `service`.

### Gaps identified

| Gap | Location | Risk |
|---|---|---|
| No log on DB insert success | `events.py:68` | Silent success — no audit trail of the insert itself |
| No log on `db_event.status` change to `failed` beyond the error log | `events.py:90` | Status transition not explicitly audited |
| `account_client.py` retry attempts not logged | `account_client.py:55-80` | Can't tell from logs whether a success was first-try or third-try |
| Account Service does not log balance computation result | `accounts.py` GET balance | No log of computed balance value — important for financial audit |
| HTTP access log (uvicorn default) and structured JSON log are separate streams | Both services | Log correlation requires two streams |

### Recommended additions

```python

        ...
```

### Suggestion 2 — Add retry logging and a failed-event background reprocessor

**Why:** Events with `status=failed` are currently stranded. In a real financial ledger, failed events need to be retried or alerted on.

```python
# Add to event-gateway/app/main.py
from contextlib import asynccontextmanager
from fastapi import FastAPI
import asyncio

@asynccontextmanager
async def lifespan(app: FastAPI):
    asyncio.create_task(retry_failed_events())
    yield

async def retry_failed_events():
    """Background task: retry status=failed events every 60 s."""
    while True:
        await asyncio.sleep(60)
        db = next(get_db())
        failed = db.query(Event).filter(Event.status == "failed").limit(10).all()
        for ev in failed:
            try:
                await account_client.apply_transaction(...)
                ev.status = "processed"
                db.commit()
            except Exception:
                pass
```
